<span data-ttu-id="f2843-118">Microsoft 365 Produkte und Dienste starke Transportprotokolle wie TLS verwenden, um zu verhindern, dass unbefugte Parteien Kundendaten abhören, während sie über ein Netzwerk übertragen werden.</span><span class="sxs-lookup"><span data-stu-id="f2843-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="f2843-119">Beispiele für Daten während der Übertragung sind E-Mail-Nachrichten, die gerade übermittelt werden, Unterhaltungen, die in einer Onlinebesprechung stattfinden, oder Dateien, die zwischen Rechenzentren repliziert werden.</span><span class="sxs-lookup"><span data-stu-id="f2843-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="f2843-122">Wie verwaltet Microsoft 365 die für die Verschlüsselung verwendeten Schlüssel?</span><span class="sxs-lookup"><span data-stu-id="f2843-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="f2843-130">Kunden mit Anforderungen zum Steuern ihrer eigenen Stammverschlüsselungsschlüssel können die Dienstverschlüsselung mit Customer Key nutzen.</span><span class="sxs-lookup"><span data-stu-id="f2843-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="f2843-131">Mit Customer Key können Kunden ihre eigenen kryptografischen Schlüssel entweder mit einem lokalen Hardware Service Module (HSM) oder Azure Key Vault (AKV) generieren.</span><span class="sxs-lookup"><span data-stu-id="f2843-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="f2843-132">Kundenstammschlüssel werden in AKV gespeichert, wo sie als Stamm einer der Schlüsselbunde verwendet werden können, die Kundendaten oder -dateien verschlüsseln.</span><span class="sxs-lookup"><span data-stu-id="f2843-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="f2843-133">Auf Kundenstammschlüssel kann nur indirekt über Microsoft 365 Dienstcode für die Datenverschlüsselung zugegriffen werden, und microsoft-Mitarbeiter können nicht direkt darauf zugreifen.</span><span class="sxs-lookup"><span data-stu-id="f2843-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
