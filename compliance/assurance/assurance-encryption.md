<span data-ttu-id="74f2f-114">Exchange Online, Microsoft Teams, SharePoint Online und OneDrive für Unternehmen verwenden auch die Dienst Verschlüsselung auf der Anwendungsebene, um Kunden Inhalte zu verschlüsseln.</span><span class="sxs-lookup"><span data-stu-id="74f2f-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="74f2f-115">Die Dienst Verschlüsselung bietet Rechte Schutz-und Verwaltungsfunktionen über einen starken Verschlüsselungsschutz hinaus.</span><span class="sxs-lookup"><span data-stu-id="74f2f-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="74f2f-116">Sie ermöglicht außerdem eine Trennung zwischen Windows-Betriebssystemen und den Kundendaten, die von diesen Betriebssystemen gespeichert oder verarbeitet werden.</span><span class="sxs-lookup"><span data-stu-id="74f2f-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="74f2f-117">Wie verschlüsseln Microsoft 365 Daten in Transit?</span><span class="sxs-lookup"><span data-stu-id="74f2f-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="74f2f-118">Microsoft 365-Produkte und-Dienste verwenden starke Transportprotokolle wie TLS, um zu verhindern, dass nicht autorisierte Personen bei der Verschiebung über ein Netzwerk auf Kundendaten abhören.</span><span class="sxs-lookup"><span data-stu-id="74f2f-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="74f2f-119">Beispiele für Daten in Transit sind e-Mail-Nachrichten, die gerade zugestellt werden, Unterhaltungen, die in einer Onlinebesprechung stattfinden, oder Dateien, die zwischen den Rechenzentren repliziert werden.</span><span class="sxs-lookup"><span data-stu-id="74f2f-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="74f2f-122">Wie verwaltet Microsoft 365 die für die Verschlüsselung verwendeten Schlüssel?</span><span class="sxs-lookup"><span data-stu-id="74f2f-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="74f2f-127">Die Dienst Verschlüsselung stellt eine zusätzliche Verschlüsselungsebene für Kundendaten bereit, die in Exchange Online, Microsoft Teams, SharePoint Online und OneDrive für Unternehmen unterstützt werden.</span><span class="sxs-lookup"><span data-stu-id="74f2f-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="74f2f-128">Die Dienst Verschlüsselung bietet Kunden zwei Optionen für die Verschlüsselungsschlüsselverwaltung: von Microsoft verwaltete Schlüssel oder Kundenschlüssel.</span><span class="sxs-lookup"><span data-stu-id="74f2f-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="74f2f-129">Bei Verwendung von von Microsoft verwalteten Tasten werden die für die Dienst Verschlüsselung verwendeten Stammschlüssel von Microsoft 365-Diensten automatisch generiert und sicher gespeichert.</span><span class="sxs-lookup"><span data-stu-id="74f2f-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="74f2f-130">Kunden mit Anforderungen zur Steuerung Ihrer eigenen Stamm Verschlüsselungsschlüssel können die Dienst Verschlüsselung mit dem Kundenschlüssel nutzen.</span><span class="sxs-lookup"><span data-stu-id="74f2f-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="74f2f-131">Mithilfe des Kunden Schlüssels können kundeneigene kryptografische Schlüssel mithilfe eines lokalen Hardware Dienst Moduls (HSM) oder eines Azure Key Vault (AKV) erstellen.</span><span class="sxs-lookup"><span data-stu-id="74f2f-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="74f2f-132">Kundenstamm Schlüssel werden in AKV gespeichert, wo Sie als Stamm einer der keychains verwendet werden können, die Kunden Postfachdaten oder-Dateien verschlüsseln.</span><span class="sxs-lookup"><span data-stu-id="74f2f-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="74f2f-133">Auf Kundenstamm Schlüssel kann nur indirekt durch Microsoft 365-Dienstcode für die Datenverschlüsselung zugegriffen werden, und es kann nicht direkt von Microsoft-Mitarbeitern darauf zugegriffen werden.</span><span class="sxs-lookup"><span data-stu-id="74f2f-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
