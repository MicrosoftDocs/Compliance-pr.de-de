## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="9f026-122">Wie verwaltet Microsoft 365 die für die Verschlüsselung verwendeten Schlüssel?</span><span class="sxs-lookup"><span data-stu-id="9f026-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="9f026-130">Kunden mit Anforderungen zur Steuerung ihrer eigenen Stammverschlüsselungsschlüssel können die Dienstverschlüsselung mit Dem Kundenschlüssel nutzen.</span><span class="sxs-lookup"><span data-stu-id="9f026-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="9f026-131">Mithilfe des Kundenschlüssels können Kunden ihre eigenen Kryptografieschlüssel mithilfe eines lokalen Hardwaredienstmoduls (Hardware Service Module, HSM) oder eines Azure Key Vault (AKV) generieren.</span><span class="sxs-lookup"><span data-stu-id="9f026-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="9f026-132">Kundenstammschlüssel werden in AKV gespeichert, wo sie als Stamm einer der Schlüsselbunde verwendet werden können, die Kundenpostfachdaten oder -dateien verschlüsseln.</span><span class="sxs-lookup"><span data-stu-id="9f026-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="9f026-133">Kundenstammschlüssel können nur indirekt über Microsoft 365-Dienstcode für die Datenverschlüsselung und nicht direkt von Microsoft-Mitarbeitern zugegriffen werden.</span><span class="sxs-lookup"><span data-stu-id="9f026-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
